    Introducción

    Esta formación de creación de ejercicios de secundaria, está centrada en conocer los aspectos que hay que tener en cuenta a la hora de crear un ejercicio. Para ello vamos a ir viendo desde los aspectos más generales, tipos de ejercicios, su visualización, o su creación desde cero de un ejercicio, teniendo como apoyo solamente la documentación que nos aporta la editorial.

    ¿Qué es un ejercicio?

    Es el medio con el cual un profesor puede evaluar los conocimientos y el progreso de sus alumnos.

    Con la llegada de las tecnologías digitales, los libros de texto de papel se están adaptando y pasando a nuevos formatos, desde lo cuales el profesor puede llevar un control más exhaustivo del progreso de sus alumnos, llegando en algunos casos a ser en tiempo real.

    En este punto es el que entramos nosotros, nuestro trabajo es ser el puente entre el formato papel y el digital, en este caso centrándonos en la digitalización de los ejercicios.

    Adaptación - Papel vs Digital

    A la hora de adaptar un ejercicio pensado para papel a formato digital, se tienen que tener en cuenta una serie de puntos.

    • Atención al detalle

      Este es el aspecto más importante a la hora de adaptar un ejercicio.

      El objetivo principal de esta adaptación es mantenerse lo más fiel posible al formato original. Para ello, tenemos que intentar mantener el estilo original, dentro de lo que sea posible, usando la misma terminología, los mismos estilos, etc. Todo ello dentro de lo que nos permitan nuestras herramientas.

    • Formato

      • El espacio con el que contamos para realizar los ejercicios es limitado.

      Tenemos dos espacios principales, el espacio de enunciado y el espacio de desarrollo del ejercicio. Cada uno de ellos tiene una posición y espacio diferente.

      • El enunciado aparece en la parte superior y es donde desarrollamos la pregunta o preguntas que va a tener que resolver el alumno.
      • La zona de desarrollo de ejercicio, es donde el alumno va a responder a las preguntas que se han planteado en el enunciado. Esta zona cuenta con distintos elementos de apoyo, los cuales veremos más adelante.
    • Tipo de respuesta

      A la hora de puntuar los ejercicios tenemos que tener en cuenta el tipo de respuesta. Vamos a trabajar con dos tipo de respuesta:

      • Ejercicios de respuesta abierta: son aquellos en los que el ejercicio no se autocorrige y es el profesor quien lo puntua.
      • Ejercicios de respuesta cerrada: son aquellos que se autocorrigen de forma inmediata por el alumnos al clickar sobre el botón de comprobar.

    Manuscritos

    Para poder realizar todos estos tipos de ejercicios, utilizamos unas plantillas de word, a las que llamamos manuscritos.

    En estos manuscritos, es donde se carga el contenido del ejercicio, enunciado, desarrollo de la pregunta, imágenes, audios, etc.

    Los manuscritos se componen de distintas cajas, cada una de ellas tiene una función diferente, las tres primeras cajas son comunes para todos los manuscritos:

    • Caja Manuscript: es donde se indica el tipo de ejercicio y la clase de formato que va a tener el ejercicio.
    • Caja Metadata: en esta caja se indica donde se encuentra el ejercicio con respecto al libro.
    • Caja Rubric: en esta caja es donde se escribe el enunciado del ejercicio.
    • La última caja básica, cambia según el formato del ejercicio, es donde se desarrolla el ejercicio.

    Subversión

    Apache Subversion (a menudo abreviado SVN, después del nombre de comando svn) es un sistema de control de versiones y revisiones de software distribuido como software libre. Los editores pueden usar Subversion para mantener versiones actuales e históricas de la documentación. SVN utiliza un modelo de servidor / cliente: esto significa que el repositorio está ubicado en un servidor remoto y el usuario necesita un programa o cliente local para descargar, confirmar y actualizar los cambios en el servidor. El cliente SVN más popular es TortoiseSVN. Una vez instalado, está totalmente integrado dentro del sistema shell de Windows

    Todos los proyectos se almacenan en un repositorio central (servidor web). Los editores mantienen una copia local del proyecto en sus ordenadores. Los editores comparten regularmente información con el servidor para mantenerla actualizada con sus copias. Cada vez que los cambios se confirman en el servidor, se genera una nueva versión de los archivos modificados. Los usuarios siempre pueden volver a la revisión previa de cada archivo si es necesario.

    Para empezar tenemos que tener instalado el Tortoise.

    Descarga Tortoise

    Operaciones básicas:

    • Checkout: Esta opción se utiliza la primera vez que los usuarios descargan una copia de un proyecto. Tenemos que conocer la URL del proyecto. Todas las URL de SVN comienzan con "svn: //".
    • Update: Esta operación se realiza cada vez que necesitamos actualizar nuestro repositorio. Ya que algún compañero a podido hacer algún cambio que afecte a nuestros archivos. Al tener el repositorio actualizado se evitan los conflictos.
    • Commit: Los editores utilizan el comando Confirmar para sincronizar sus contenidos locales en el repositorio en el lado del servidor y compartirlos con otros editores. En otras palabras, cuando un editor realiza una serie de cambios en un repositorio, debe de compartirlos, comitear, para que el resto de compañeros tengan acceso a cambio realizado.
    • Revert: Sirve para revertir el cambio realizado en los archivos del repositorio, se tiene que realizar antes de que esos cambios sean comiteados.
    • Resolve: En algunas ocasiones surgen conflictos entre los archivos locales de un editor, ya que puede haber un archivo que no se haya actualizado.
    • Clean up: Sirve para hacer una limpieza más a fondo del repositorio, eliminando los archivos que no sean correctos.

    Creación de ejercicios

    Una vez ya conocemos los tipos de ejercicios que podemos crear y las distintas visualizaciones que podemos utilizar, vamos a crear un ejercicio desde cero.

    El tipo de ejercicio: lo más fácil para decidir si escoger un tipo de ejercicio u otro es intentar ser lo más fiel posible al libro de texto que tenemos que adaptar. En algunas ocasiones no es posible y tenemos que adaptar el ejercicio a las herramientas de las que disponemos.

    En el video vamos a crear un ejercicio desde el principio. Pudiendo observar la nomenclatura básica a la hora de crear un ejercicio.

    Nomenclatura específica en la creación de ejercicios:

    • [] - Para crear una incógnita usamos los corchetes, exceptuando los ejercicios que llevan un tipo de manuscrito concreto, como puede ser un crucigrama. La tipología de ejercicios que usan esta nomenclatura son: Keying, Dropdown, Checkbox, Click Drag and drop y Reorder.
    • [opción 1|opción 2] - Dentro de los ejercicios que usan los corchetes para crear la incógnita, todos aquellos que son de seleccionar una opción u otra, llevan una línea vertical, | creada con altgr + 1, que separa las distintas opciones.

      Otro uso de la línea vertical se da en los ejercicios de tipo keying. En este tipo de ejercicios se usa para dar distintas opciones en una misma incógnita. Se puede dar el caso en el que queramos permitir al alumno que una misma palabra se escriba de formas diferentes, por ejemplo [Caballo|caballo| CABALLO]. Al escribirlo de esta manera, da igual como se escriba la palabra, siempre dará una de estas opciones como válida.

    • [texto*] - En los ejercicios que tienen distintas opciones, para marcar cual es la correcta se utiliza el asterisco *.
    • [texto+] - En los ejercicios en los que necesitemos que una de las respuestas se muestre a modo de ejemplo se usa el +
    • [(group:a):][(group:a):] - Dentro de los corchetes se pueden incluir algunas clases para que modifiquen su comportamiento. En este caso al utilizar la clase: group, las distintas palabras que vayan dentro de un mismo grupo, se resuelven en común. Se utiliza en ejercicios de keying, para poder escribir dos palabras diferentes en dos posiciones y las dos sean correctas. También se usa en los drag and drop, para que el drag depositado se pueda poner en distintas posiciones.
    • [(evaluate:false):] - Es una clase que se utiliza para convertir el ejercicio en abierto y no tenga puntuación.
    • {U01_02_03.png} o {U01_02_03.mp3} - Este tipo de nomenclatura se usa para incluir un audio o una imagen en el ejercicio.
    • {model-answer:Respuesta modelo} - Esta clase se usa para incorporar una respuesta modelo.
    • [\2 + 3\] - En algunos casos es necesario utilizar símbolos que no funcionan ya que tienen otra función, para solucionar este problema, se usan las dos barras \texto\ altgr + º, para que se puedan mostrar. En este caso el símbolo +.
    • Para poder utilizar distintos tipos de ejercicios en un solo manuscrito, tenemos que escribir dentro de los corchetes la tipología del ejercicio. Un ejemplo sería, dentro de un documento keying, si queremos que aparezca un dropdown, se tendría que escribir lo siguiente dentro de los corchetes: [dropdown: opción 1|opción 2].

    Manipulación del contenido

    El contenido que va en las cajas de enunciado o en la de desarrollo de ejercicio, puede ser modificado para que se parezca lo máximo al original.

    Se pueden usar diferentes elementos:

    • Etiquetas html: aunque usemos documentos de word para la carga de contenido, al transformarlos en contenido digital algunos elementos pueden dar problemas, algunos casos son la negrita o la cursiva, que en algunas ocasiones producen espacios no deseados. Para solucionar este problema usamos dos etiquetas html, para escribir en negrita, el texto va dentro de la etiqueta <b>texto</b>, el resultado sería: ejemplo. Si el texto tiene que ir en cursiva usamos la etiqueta <i>texto</i>, el resultado sería: ejemplo. Además de estas dos etiquetas se pueden utiliza
    • Entidades html: Son aquellos elementos que tienen una forma de escribirse muy concreta y que el manuscrito convierte en símbolos o letras que necesitamos y no se pueden escribir con el teclado. Un ejemplo sería &rarr;, este código equivale a una flecha hacia la derecha, el resultado sería: →.
    • Estilo en línea: este tipo de estilos se deben usar lo menos posible, un ejemplo, sería <span style="color:blue">Texto</span> el resultado sería: Texto.
    • También existen unas etiquetas, denominadas clases, que se han creado de forma específica para que funcione en estos manuscritos. Se han creado clases, para poder escribir fracciones, radicales, etc.

    Photoshop

    A la hora de editar las imágenes se pueden dar dos situaciones:

    Que ya tengamos las imágenes recortadas o que tengamos que recortarlas.

    Si la imagen ya está recortada, solo habría que darle el tamaño adecuado.

    Para realizar estas acciones vamos a usar unos comandos básicos de photoshop: cambiar tamaño y recortar.

    • Si la imagen ya nos la han dado recortada, solo hay que cambiar su tamaño, para ello podemos ir al menú de arriba, pinchar en imagen y seleccionar Tamaño de imagen o pulsar ctrl + alt + I. Nos abrirá un diálogo donde pondremos el tamaño que se adecue más al ejercicio.
    • Si la imagen la tenemos que obtener del pdf, tendríamos que dar algún paso más: se hace un pantallazo al pdf, se pega en un nuevo documento de photoshop, ctrl + N, y lo recortaremos, seleccionamos la imagen que queremos, pulsando la M, una vez seleccionada la imagen la recortamos, pulsando C y enter. Una vez recortado solo habría que cambiar el tamaño y guardar.
    • Una vez obtenida la imagen deseada, pasamos a guardarla. Para guardarla podemos ir al menú Archivo y seleccionar guardar para página web o pulsar Ctrl + Alt + Shift + S. En el diálogo que aparece seleccionamos png-24, clickeamos en salvar y seleccionamos la carpeta donde tiene que ir la imagen.
